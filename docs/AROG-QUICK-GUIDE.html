How to Setup @arog on Any New Repo or Project

5-minute integration process

1

Copy AROG Folders

Copy the complete AROG framework to your new repository:

# Copy from AROG repository
cp -r /Users/arog/Learn/arog/.arog/ /your-new-project/
cp -r /Users/arog/Learn/arog/.github/ /your-new-project/

# Navigate to your project
cd /your-new-project/

‚ö†Ô∏è Important: Use Root Folders, Not Integration Kit

Always copy from /Users/arog/Learn/arog/.arog/ and /Users/arog/Learn/arog/.github/ (root folders), as they contain the latest features including alert system, Playwright MCP, and operation logger. The integration kit is slightly outdated.

2

Install Dependencies

Install AROG dependencies and auto-configure:

# Install AROG dependencies (auto-launches interactive CLI)
cd .arog
npm install

# This automatically:
# ‚úì Sets up Playwright MCP server
# ‚úì Launches interactive CLI welcome screen
# ‚úì Configures all automation tools
3

Configure Environment (Optional)

Customize AROG for your team's specific needs:

# Edit alert configuration
nano .arog/config/alerts.config.json

# Add your team's email/WebEx details
# Configure routing rules
# Set notification preferences

# Edit GitHub Copilot instructions
nano .github/copilot-instructions.md

# Customize project-specific guidance
4

Validate Installation

Run health check to ensure everything works:

# From project root
npm run arog:health

# Should show:
# ‚úÖ Node.js Version
# ‚úÖ npm Version
# ‚úÖ Git Repository
# ‚úÖ package.json
# ‚úÖ GitHub Workflows
# ‚úÖ AROG Agent
5

Start Using @arog!

You're ready! Start automating everything:

# In VS Code, ask @arog:
@arog what can you do?
@arog review this code
@arog run tests
@arog deploy to staging

# Or use interactive CLI:
npx arog

# Or run specific commands:
npm test              # Run all tests
npm run lint          # Check code quality
npm run test:e2e      # E2E tests

‚úÖ That's It! Total Time: Less than 5 Minutes

AROG is now running in your project with full automation active. All GitHub workflows will trigger automatically on commits, PRs, and deployments.
