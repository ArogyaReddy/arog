ğŸ¢ org-config.json Level 1 - Organization

What It Is:

Company-wide standards that apply to ALL projects in your organization.

When to Use:

  • CTO/Leadership wants consistent standards across all teams
  • Security team mandates vulnerability thresholds
  • Compliance requires specific accessibility standards
  • Finance team enforces cost optimization

How to Use:

{
  "standards": {
    "codeQuality": {
      "eslint": "airbnb",
      "prettier": true,
      "typescript": "strict"
    },
    "testing": {
      "minCoverage": 80
    },
    "security": {
      "vulnerabilityThreshold": "moderate"
    },
    "accessibility": {
      "wcagLevel": "AA"
    }
  }
}

Result: All 30 projects now follow the same code quality, testing, security, and accessibility standards!

How @arog Benefits:

  • Enforces company-wide quality standards automatically
  • Ensures all projects meet security requirements
  • Guarantees accessibility compliance (WCAG 2.1 AA)
  • Optimizes AI costs (70-85% savings) organization-wide
  • Provides consistent developer experience across all teams

What It Is:

Team-specific rules that override organization config. Different teams can have different tech stacks and requirements.

  • Frontend team needs React-specific linting rules
  • Backend team requires API testing frameworks
  • Mobile team has platform-specific standards
  • Team lead wants higher coverage than org minimum

    "testing": {
      "minCoverage": 90
    },
    "codeReview": {
      "minReviewers": 2
    }
  },
  "teamSpecific": {
    "techStack": ["TypeScript", "React", "Next.js"]
  }
}
